# Estructura de un commit

[tipo][alcance]: [asunto][descripción]

## Tipo:

Indica lo que se hace en el archivo subido: fix/test/refactor/revert/build, etc

## Alcance

Indica el archivo donde se ha hecho el cambio o corrección

## Asunto

Es una nota legible (>50 caracteres) en presente simple (declarativa) en la que se indica la modificación concreta que se ha realizado

## Descripción 

Responde al por qué y al cómo del cambio. Descripción que empieza con un vb en presente simple en inglés
